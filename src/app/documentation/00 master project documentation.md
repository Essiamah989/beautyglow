## 🎯 QUICK PROJECT OVERVIEW

**What:** Multi-tenant SaaS platform providing professional websites with online booking for Tunisian beauty salons

**Problem:** 80% of Tunisian beauty salons lack professional online presence, rely only on social media, lose customers due to missed calls and manual booking chaos

**Solution:** 5-minute setup to get professional website (subdomain.beautyglow.tn) with integrated booking system

**Pricing:**
- Basic: 89 TND/month
- Pro: 149 TND/month
- Elite: 229 TND/month
- 14-day free trial on ALL plans

**Target Market:** 50,000+ beauty salons in Tunisia, focusing initially on urban areas (Tunis, Sfax, Sousse)

**Tech Stack:** Next.js 14 + Supabase + Vercel + TypeScript + Tailwind CSS

**Timeline:** 4 months to launch with 30+ active salons (currently end of Month 2)

**Revenue Goal Year 1:** 500 paying customers — target MRR depends on plan mix

## 🔑 KEY TECHNICAL DECISIONS

**Auth client:** Must use `createBrowserClient` from `@supabase/ssr` — NOT `createClient` from `@supabase/supabase-js`. The SSR package correctly handles cookies for Next.js middleware.

**Theme system:** All 4 CSS themes imported statically in SalonClient.tsx. Active theme controlled via `data-theme="[themename]"` on root div. Each CSS file scopes rules under `[data-theme="themename"]`.

**Multi-tenancy:** Every table has `business_id` FK. RLS policies ensure complete data isolation. Never query without `business_id` filter.

**Email:** Resend for transactional email. Dev env limited to 2 emails/hour — turn off email confirmation in Supabase for dev testing.

**Subdomain routing:** Middleware extracts subdomain → queries Supabase → rewrites to `/sites/[businessId]`. Main domain shows landing page.
